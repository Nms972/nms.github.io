Diagramme UML

Le diagramme ci-dessus représente la structure des principales classes de l'application, ainsi que leurs relations. Voici une description des éléments principaux :

CLIConsole :

Cette classe gère l'interaction en ligne de commande avec l'utilisateur. Elle propose des méthodes pour afficher les dossiers, les fichiers, les statistiques, et pour sauvegarder ou comparer des snapshots.

GUI :

Classe responsable de l'interface graphique de l'application. Elle contient les méthodes nécessaires pour afficher les fichiers, les images et les métadonnées dans une interface utilisateur, tout en permettant la navigation dans les répertoires et la gestion des snapshots.

GUI :

Classe responsable de l'interface graphique de l'application. Elle contient les méthodes nécessaires pour afficher les fichiers, les images et les métadonnées dans une interface utilisateur, tout en permettant la navigation dans les répertoires et la gestion des snapshots.

Dossier :

Représente un dossier contenant des fichiers. Elle inclut des méthodes pour accéder à la liste des fichiers, leurs métadonnées et identifier le type principal de contenu.

Dossier :

Représente un dossier contenant des fichiers. Elle inclut des méthodes pour accéder à la liste des fichiers, leurs métadonnées et identifier le type principal de contenu.

Fichier :

Cette classe modélise un fichier individuel. Elle contient des attributs comme le chemin, la taille, et le type de fichier, ainsi qu'une méthode pour récupérer une instance de l'objet Image associée.

Image :

Classe représentant une image avec des attributs tels que ses dimensions et sa résolution.

Snapshot :

Représente un état capturé d’un ensemble de fichiers ou d’un dossier à un moment donné. Les snapshots contiennent des métadonnées telles que la date et l’heure de capture.

SnapshotStock :

Classe qui centralise et gère les snapshots. Elle permet d'ajouter, de sauvegarder, de charger et d'extraire des snapshots au format fichier.

Statistique :

Cette classe est dédiée à l’analyse des données, comme le nombre total de fichiers et d’images, ainsi que la taille moyenne des fichiers.

Métadonnée :

Représente des informations supplémentaires associées aux fichiers, telles que l'auteur, la description et les dimensions.

Diagramme de cas d'utilisation

Ce diagramme représente les différentes fonctionnalités de l'application et les interactions avec l'utilisateur. Le diagramme est organisé autour de plusieurs cas d'utilisation, chacun représentant une fonctionnalité ou un objectif que l'utilisateur peut accomplir. Voici les principaux cas d'utilisation :

Utilisateur (Acteur principal):

L'acteur principal, représenté par un bonhomme est l'Utilisateur. Cet acteur interagit avec l'application pour accomplir diverses tâches.

Accéder au dossier :

Permet à l'utilisateur d'accéder à un dossier spécifique sur le système de fichiers.

Extends :

Ce cas inclut plusieurs actions supplémentaires qui peuvent être réalisées une fois que l'utilisateur a accédé au dossier, telles que Extraire des statistiques et Sauvegarder l'état d'un répertoire.

Accéder au fichier_image :

L'utilisateur peut accéder à un fichier image spécifique, ce qui déclenche les actions suivantes :

  • Afficher l'image : L'image est affichée à l'utilisateur.
  • Afficher les métadonnées : Affichage des informations associées à l'image, comme la taille, le format, etc.
  • Extraire les métadonnées : Permet à l'utilisateur d'extraire des informations détaillées sur l'image, telles que la date de création et d'autres attributs.

Cas d'utilisation secondaires et extensions :

  • Lister les fichiers : Affiche une liste de fichiers dans le dossier.
  • Extraire des statistiques : Permet de visualiser des statistiques concernant les fichiers, comme le nombre total d'images ou le nombre d'images par format. Ce cas inclut des actions comme compter le nombre total d'images et Identifier le nombre d'images par format.
  • Rechercher des fichiers : Recherche des fichiers dans un répertoire en fonction de certains critères, incluant des Critères de recherche spécifiques.
  • Miniature : Affiche une miniature d'une image pour faciliter la navigation et l'identification visuelle rapide des fichiers.
  • Zoomer sur une image : Une extension du cas d'utilisation Afficher l'image, permettant à l'utilisateur de zoomer sur l'image pour une vue plus détaillée.
